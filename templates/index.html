DICT

This is a web based, crud automation system that automates examinee data. start now.

Powered by:

Database Operations:

The application connects to the MySQL database to perform various operations such as user registration, user login, displaying users, updating users, and managing examinee records.

HTML Templates:

The application renders HTML templates for different routes to provide a user interface. These templates are responsible for displaying data and forms.

Session Management:

Session Management: The application uses the session to manage user authentication. Users are required to log in before accessing certain routes.

Error Handling:

The application handles exceptions and displays flash messages to inform users about the success or failure of operations.

Running the Application:

The application runs in debug mode if the script is executed directly.

Routes:

The application defines various routes for different functionalities:
/: The home route, which is protected by the login_required decorator, displays the main page.
/booking: Displays a booking page (which seems to be related to booking examinations).
/congratulatory_message: Displays a congratulatory message page.
/logout: Logs the user out by popping the 'username' from the session.
/search: Handles a POST request to search for examinees based on a search query and displays the results.
/booking_process: Handles the booking process when a user submits a form with examinee information. It inserts data into the appropriate table based on the selection and checks if the examinee passed the examination.
/examinees: Displays a list of examinees. It allows filtering by various criteria and supports pagination.
/examinees/passed: Displays a list of examinees who passed the examination (filtered).
/login: Displays a login page and handles user login attempts.
/login_process: Handles the login process by checking the user's credentials and setting a session variable if the login is successful.
/register: Displays a registration page and handles user registration.
/display: Displays a table of registered users.
/delete_process//: Deletes a user based on their user ID.
/update_process_one//: Displays a form to update user information based on user ID.
/update_process_two: Handles the user information update process.
/examinee_delete//: Deletes an examinee based on their ID.
/examinee_update_one//: Displays a form to update examinee information based on ID.
/examinee_update_two: Handles the examinee information update process.
